# Práctica III: Desarrollo Dirigido por Tests (TDD)

Este proyecto implementa un sistema de gestión de tareas siguiendo la metodología **TDD** (*Test-Driven Development*).Se centra
en el uso de herramientas profesionales como Maven para la gestión de dependencias y JUnit para la validación de software.
## Metodología TDD
La metodología TDD (Test-Driven Development o Desarrollo Guiado por Pruebas) es un enfoque de ingeniería de software donde se escriben las pruebas automatizadas (generalmente unitarias) 
antes de desarrollar el código funcional. Su ciclo fundamental consiste en tres pasos: Red (escribir un test que falle), Green (hacer que pase con el mínimo código) y Refactor (mejorar el código). 

**Beneficios principales:**
* **Mayor calidad:** Código más robusto y menos propenso a errores.
* **Reducción de mantenimiento:** Menos bugs y mejor diseño (código limpio).
* **Confianza:** Permite cambiar el código con seguridad, sabiendo que los tests detectarán regresiones.
* **Documentación:** Las pruebas actúan como documentación viva del comportamiento del sistema
